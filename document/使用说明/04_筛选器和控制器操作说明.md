# 筛选器和控制器操作说明

## 1. 页面布局

大盘页面从上到下分为四个区域：

```
┌───────────────────────────────────────────┐
│  标题区：App名 - 多时间维度ROI趋势         │
│          副标题 / 数据范围说明             │
├───────────────────────────────────────────┤
│  筛选器区：4 个下拉菜单                   │
├───────────────────────────────────────────┤
│  控制器区：2 组单选按钮                   │
├───────────────────────────────────────────┤
│  图表区                                   │
└───────────────────────────────────────────┘
```

---

## 2. 筛选器（Filter Bar）

筛选器区域包含 4 个下拉选择框，用于限定图表展示的数据范围。

### 2.1 用户安装渠道

| 属性 | 说明 |
|---|---|
| 类型 | 静态下拉 |
| 可选值 | `Apple`、`Google Play`、`All` |
| 后端影响 | **无**（当前版本数据库中无渠道字段，仅 UI 展示） |
| 说明 | 预留字段，用于后续扩展多渠道数据支持 |

### 2.2 出价类型

| 属性 | 说明 |
|---|---|
| 类型 | 动态下拉（从数据库读取） |
| 来源 | `GET /api/roi/filters` → `bid_types[]` |
| 当前可选值 | `CPI` |
| 后端影响 | ✓ 作为 `bid_type` 参数传入趋势 API |

### 2.3 国家地区

| 属性 | 说明 |
|---|---|
| 类型 | 动态下拉（从数据库读取） |
| 来源 | `GET /api/roi/filters` → `country_regions[]` |
| 当前可选值 | `美国`、`英国` |
| 后端影响 | ✓ 作为 `country_region` 参数传入趋势 API（URL 编码） |

### 2.4 APP

| 属性 | 说明 |
|---|---|
| 类型 | 动态下拉（从数据库读取） |
| 来源 | `GET /api/roi/filters` → `apps[]` |
| 当前可选值 | `App-1`、`App-2`、`App-3`、`App-4`、`App-5` |
| 后端影响 | ✓ 作为 `app` 参数传入趋势 API；同时更新页面标题 |

### 操作说明

1. 页面加载时，动态选项自动从 API 获取并填充
2. 默认选中各选项的**第一项**
3. **任意一个筛选器变更**都会立即触发新的 API 调用，图表自动刷新

## 3. 控制器（Options Bar）

控制器区域包含 2 组单选按钮，控制图表的**显示模式**，不影响数据范围。

### 3.1 数据显示模式

| 选项 | 值 | 说明 |
|---|---|---|
| ○ 显示移动平均值（默认）| `ma` | 对原始数据应用 7 日简单移动平均（SMA），平滑短期波动 |
| ○ 显示原始数据 | `raw` | 直接展示数据库中的原始 ROI 值，不做平滑处理 |

**移动平均算法说明：**

```
MA[i] = mean(values[max(0, i-6) .. i])
```

- 窗口大小：7 天（trailing window）
- 每个 ROI 指标独立计算
- 在后端计算后返回，通过 `moving_avg=true/false` 参数控制

**页面副标题对应变化：**

| 模式 | 副标题 |
|---|---|
| 移动平均值 | `(7日移动平均)` |
| 原始数据 | `(原始数据)` |

**图例名称对应变化：**

| 模式 | 图例示例 |
|---|---|
| 移动平均值 | `当日(7日均值)`, `1日(7日均值)` … |
| 原始数据 | `当日`, `1日` … |

### 3.2 Y 轴刻度

| 选项 | 值 | 说明 |
|---|---|---|
| ○ 线性刻度 | `linear` | Y 轴按等差间隔显示，数值变化幅度直观 |
| ○ 对数刻度（默认）| `log` | Y 轴按对数分布显示，适合数量级差异大的数据 |

**对数刻度下的 Y 轴刻度值：**

```
1%  3%  7%  10%  30%  70%  100%  300%  500%  1000%
```

**切换 Y 轴刻度不会触发 API 重新调用**，仅在前端重新渲染图表。

## 4. 操作流程示例

### 场景 1：查看 App-3 在英国的 90 日 ROI 趋势

1. 在 **国家地区** 下拉框中选择 `英国`
2. 在 **APP** 下拉框中选择 `App-3`
3. 保持其他选项默认（出价类型 `CPI`，显示移动平均值，对数刻度）
4. 图表自动刷新，标题变为 `App-3 - 多时间维度ROI趋势`

### 场景 2：对比移动平均与原始数据的差异

1. 先在 **数据显示模式** 中选择 `显示移动平均值`，观察图表曲线
2. 切换至 `显示原始数据`，观察曲线波动幅度变化
3. 移动平均模式下曲线更平滑，原始数据模式下短期波动更明显

### 场景 3：观察低 ROI 区间的细节

1. 在 **Y轴刻度** 中选择 `对数刻度`（默认）
2. 低 ROI 区间（0%~30%）和高 ROI 区间（100%~500%）均清晰可见
3. 如需强调高低区间的绝对差距，切换至 `线性刻度`

### 场景 4：查看 0% 的具体含义

1. 将鼠标悬停在图表上接近最新日期的数据点
2. Tooltip 中若某指标显示 `0% (日期不足)` → 该日期距今不足对应统计天数，ROI 尚未产生
3. 若显示 `0% (真实0%)` → 该期间确实无收益

## 5. 控制器与 API 参数对照

| 控制器 | 对应 API 参数 | 影响 |
|---|---|---|
| 出价类型 下拉 | `bid_type` | 重新请求 API |
| 国家地区 下拉 | `country_region` | 重新请求 API |
| APP 下拉 | `app` | 重新请求 API |
| 数据显示模式 | `moving_avg=true/false` | 重新请求 API |
| Y 轴刻度 | — | 仅前端重渲染 |
| 用户安装渠道 | — | 无效果（预留） |
