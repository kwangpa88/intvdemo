# 前端组件架构说明

## 1. 技术框架

| 技术 | 说明 |
|---|---|
| Next.js 14 App Router | 基于文件系统的路由，所有页面位于 `app/` 目录 |
| React 18 | `useState` / `useEffect` / `useCallback` Hooks |
| TypeScript | 全量类型标注 |
| TailwindCSS | 原子化 CSS，无自定义 CSS 文件（除全局变量） |
| ShadCN/UI | Button 组件基础（已移除 `@radix-ui/react-slot` 依赖） |
| Recharts | LineChart 图表渲染 |

---

## 2. 路由结构

```
app/
├── layout.tsx          ← 全局布局（<html><body>）
├── globals.css         ← CSS 变量（颜色令牌）+ Tailwind base
├── page.tsx            ← / → redirect('/dashboard')
├── dashboard/
│   └── page.tsx        ← /dashboard  ROI 趋势大盘（主界面）
└── helloworld/
    └── page.tsx        ← /helloworld  Hello World 示范页
```

## 4. 子组件说明

### 4.1 `SelectField`

封装下拉选择框的通用组件。

| Prop | 类型 | 说明 |
|---|---|---|
| `label` | `string` | 选框上方标签文字 |
| `value` | `string` | 当前选中值 |
| `options` | `string[]` | 选项列表 |
| `onChange` | `(v: string) => void` | 变更回调 |

### 4.2 `RadioGroup`

单选组组件，带自定义圆圈样式（替代原生 radio 样式）。

| Prop | 类型 | 说明 |
|---|---|---|
| `title` | `string` | 组标题 |
| `options` | `{ value: string; label: string }[]` | 选项列表 |
| `value` | `string` | 当前选中 value |
| `onChange` | `(v: string) => void` | 变更回调 |

### 4.3 `CustomTooltip`

Recharts 自定义 Tooltip，增强 0% 值的语义说明。

| Prop | 类型 | 来源 |
|---|---|---|
| `active` | `boolean` | Recharts 注入 |
| `payload` | `PayloadItem[]` | Recharts 注入（含 `payload` 原始点数据） |
| `label` | `string` | 当前悬停 date 值 |

**0% 区分逻辑**

每个 `ChartPoint` 上，若某 ROI 指标原始值为 `0`，会附加 `dayX_roi__zero` 元数据：

| 元数据值 | 含义 | Tooltip 显示 |
|---|---|---|
| `'insufficient'` | 截止日期距该日期不足统计周期 | `0%` + 黄色 `(日期不足)` |
| `'real'` | 该日期 ROI 确实为 0（无收益） | `0%` + 红色 `(真实0%)` |

## 6. 数据处理函数

### `buildChartData(data, forecast) → ChartPoint[]`

将后端返回的 `data[]` 和 `forecast[]` 合并为 Recharts 可用的单一数组。

- 实际数据点：包含 `dayX_roi` 字段，预测字段设为 `null`
- 预测数据点：包含 `fX` 字段，实际字段设为 `null`
- 桥接点（Bridge）：最后一个实际点同时填充第一个预测值，使两者曲线连续
- 0% 标记：原始值为 `0` 时，追加 `dayX_roi__zero: 'insufficient' | 'real'`

### `getZeroReason(date, cutoffDate, periodDays)`

判断 0% 的成因：

```
diffDays = cutoffDate - date（天数）
diffDays < periodDays  →  'insufficient'（日期不足）
diffDays >= periodDays →  'real'（真实0%）
```

## 8. Next.js 配置（API 代理）

`frontend/next.config.js` 中配置了 rewrites，将前端 `/api/*` 请求转发至后端：

```javascript
async rewrites() {
  return [{
    source: '/api/:path*',
    destination: 'http://localhost:4000/api/:path*',
  }]
}
```

这使得前端无需在 `fetch` 中写入后端地址，也避免了浏览器层面的 CORS 问题。
