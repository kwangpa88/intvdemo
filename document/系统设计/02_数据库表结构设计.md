# 数据库表结构设计

## 1. 概述

数据库采用 **SQLite**，文件路径为 `backend/app_roi.db`，使用 Node.js 内置 `node:sqlite`（`DatabaseSync`）模块进行同步访问。

---

## 2. 表：`app_roi`

### 2.1 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS app_roi (
  id             INTEGER PRIMARY KEY AUTOINCREMENT,
  date           DATE         NOT NULL,
  app            VARCHAR(255) NOT NULL,
  bid_type       VARCHAR(255) NOT NULL,
  country_region VARCHAR(255) NOT NULL,
  total_installs INT          NOT NULL,
  day0_roi       FLOAT,
  day1_roi       FLOAT,
  day3_roi       FLOAT,
  day7_roi       FLOAT,
  day14_roi      FLOAT,
  day30_roi      FLOAT,
  day60_roi      FLOAT,
  day90_roi      FLOAT
);
```

### 2.2 列说明

| 列名 | 类型 | 非空 | 说明 | 对应 CSV 原始列名 |
|---|---|---|---|---|
| `id` | INTEGER PK AUTOINCREMENT | ✓ | 自增主键 | — |
| `date` | DATE | ✓ | 数据日期，格式 `YYYY-MM-DD` | `日期`（去除星期后缀） |
| `app` | VARCHAR(255) | ✓ | 应用名称（如 App-1） | `app` |
| `bid_type` | VARCHAR(255) | ✓ | 出价类型（如 CPI） | `出价类型` |
| `country_region` | VARCHAR(255) | ✓ | 国家/地区（如 美国、英国） | `国家地区` |
| `total_installs` | INT | ✓ | 当日应用安装总次数 | `应用安装.总次数` |
| `day0_roi` | FLOAT | — | 当日 ROI（%，去除 `%` 符号） | `当日ROI` |
| `day1_roi` | FLOAT | — | 1 日 ROI | `1日ROI` |
| `day3_roi` | FLOAT | — | 3 日 ROI | `3日ROI` |
| `day7_roi` | FLOAT | — | 7 日 ROI | `7日ROI` |
| `day14_roi` | FLOAT | — | 14 日 ROI | `14日ROI` |
| `day30_roi` | FLOAT | — | 30 日 ROI | `30日ROI` |
| `day60_roi` | FLOAT | — | 60 日 ROI | `60日ROI` |
| `day90_roi` | FLOAT | — | 90 日 ROI | `90日ROI` |

> **ROI 值存储说明**：CSV 中的 `6.79%` 存储为 `6.79`（浮点数，不含 `%`）。

### 2.3 数据示例

| id | date | app | bid_type | country_region | total_installs | day0_roi | day1_roi | day7_roi | day30_roi | day90_roi |
|---|---|---|---|---|---|---|---|---|---|---|
| 1 | 2025-04-13 | App-1 | CPI | 美国 | 4849 | 6.79 | 14.24 | 64.44 | 214.65 | 413.81 |
| 2 | 2025-04-13 | App-2 | CPI | 美国 | 2180 | 4.19 | 14.74 | 35.63 | 107.64 | 206.66 |

## 3. CSV → 数据库 字段映射

| CSV 原始标题 | 处理方式 | 数据库列 |
|---|---|---|
| `日期`（如 `2025-04-13(日)`） | 去除 `(X)` 星期后缀 | `date` |
| `app` | 直接映射 | `app` |
| `出价类型` | 直接映射 | `bid_type` |
| `国家地区` | 直接映射 | `country_region` |
| `应用安装.总次数` | `parseInt()` | `total_installs` |
| `当日ROI`（如 `6.79%`） | 去除 `%`，`parseFloat()` | `day0_roi` |
| `1日ROI` ~ `90日ROI` | 同上 | `day1_roi` ~ `day90_roi` |

## 4. 查询模式

### 4.1 获取筛选项

```sql
SELECT DISTINCT app            FROM app_roi ORDER BY app;
SELECT DISTINCT bid_type       FROM app_roi ORDER BY bid_type;
SELECT DISTINCT country_region FROM app_roi ORDER BY country_region;
```

### 4.2 按条件查询近 N 日趋势（AVG 聚合）

```sql
SELECT
    date,
    AVG(day0_roi)  AS day0_roi,
    AVG(day1_roi)  AS day1_roi,
    AVG(day3_roi)  AS day3_roi,
    AVG(day7_roi)  AS day7_roi,
    AVG(day14_roi) AS day14_roi,
    AVG(day30_roi) AS day30_roi,
    AVG(day60_roi) AS day60_roi,
    AVG(day90_roi) AS day90_roi
FROM app_roi
WHERE app = :app
  AND bid_type = :bid_type
  AND country_region = :country_region
  AND date >= (
    SELECT date(MAX(date), '-' || :days || ' days')
    FROM app_roi
    WHERE app = :app AND bid_type = :bid_type AND country_region = :country_region
  )
GROUP BY date
ORDER BY date;
```

> `AVG()` 聚合是为了应对同一日期可能存在多条记录的情况。

## 5. 数据库初始化逻辑

初始化入口：`backend/src/db.ts` → `initAppRoiTable(dropIfExists: boolean)`

| 参数 | 行为 |
|---|---|
| `dropIfExists = false`（默认） | `CREATE TABLE IF NOT EXISTS`，表已存在则跳过 |
| `dropIfExists = true` | 先 `DROP TABLE IF EXISTS`，再重新创建 |

环境变量控制：

```bash
# 销毁并重建表（慎用，数据会丢失）
DROP_TABLE=true npm run dev
```
