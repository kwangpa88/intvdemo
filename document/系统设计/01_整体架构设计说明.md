# 整体架构设计说明

## 1. 系统概述

本系统是一个 **App ROI 多时间维度趋势分析平台**，用于可视化展示移动应用在不同国家/地区、不同出价类型下的 ROI 回报率随日期变化的趋势，支持 7 日移动平均、对数/线性 Y 轴切换以及 14 日预测功能。

## 5. 数据流

```
用户操作(筛选器变更)
        │
        ▼
fetchTrend() [前端 useCallback]
        │
        ▼
GET /api/roi/trend?app=&bid_type=&country_region=&days=90&moving_avg=true
        │ (Next.js rewrite proxy)
        ▼
Express Router → roi.ts
        │
        ▼
SQLite SELECT + GROUP BY date + AVG()
        │
        ▼
JavaScript: 7日移动平均 OR 原始数据
        │
        ▼
JavaScript: 线性回归 → 14日预测
        │
        ▼
JSON 响应 { data[], forecast[] }
        │
        ▼
buildChartData() → 合并 actual + forecast + 0%原因元数据
        │
        ▼
Recharts LineChart 渲染
```

## 6. 关键设计决策

| 决策 | 原因 |
|---|---|
| 使用 Node.js 内置 `node:sqlite` | 避免第三方 native 依赖，Node.js 22+ 内置支持 |
| Next.js rewrite 代理 `/api/*` | 前端与后端同源，避免 CORS 复杂配置 |
| 移动平均在后端计算 | 减少前端计算压力，保持 API 数据语义清晰 |
| 预测在后端计算 | 线性回归逻辑集中管理，便于后续替换算法 |
| 0% 原因区分在前端判断 | 该逻辑依赖切断日期，前端掌握完整数据上下文 |
